1.0 这个是干什么的？

首先,你看到的这篇文章,不是一篇Linux的练级攻略本,也不是一篇快速入门Linux的技术文章,而是一篇笔者自己安装使用linux的游记,笔者做这个的想法,只是想稍微介绍一下Linix,而且可以尽可能减少入门的难度. 因为一般情况下,直接入手Linux的道路会相当曲折

如何阅读呢？本文的大部分内容都是可以直接【跟着做】的,笔者会尽量保证文档的更新,而且会尽可能的考虑到所有的情况,一般情况下,出入不会很大,你可以跟随着我的思路过程,接触了解Linux,同时,大部分的内容,我都提供了更加深入的内容,如果你有感兴趣可以进行更多的阅读和学习,得到的收获自然也是不同的.附加的内容,都以超链接的形式标注在文档上面.

其次,希望读者可以有培养一种先搜后问的态度,你所看到的大部分内容,都是笔者集合了大部分互联网上的优秀资源总结出来的,,当然有问题不解的可以给我发邮件：emlcjnil#gmail.com, 也欢迎各路大牛指证.

为什么是Linux？Linux是开源的,是免费的,同时也是充满魅力的,关于win的糟糕可以查看 Windows编程革命简史, 而学习Linux可以获得各种各样的收获,这里就不赘述Linux的好处了.同样,这篇文档里面所提到的软件方法,都是可以“免费”得到的,不会鼓动大家去下载盗版软件.

同时,本文档尊崇开源的精神,你可以将本文档的东西转载或发表到任何地方,但是希望能够注明.本站所发表的每一张图片都是原创的,笔者并没有加上优雅(且简单)的水印,是希望这种崇尚自由的精神贯穿其中.

最后,为了加强文档的有趣性,笔者会加上适当的感悟或者说是吐槽,为了不影响文档的内容,这一部分会以小字出现.

1.1 安装之前的准备

两个问题

本文是针对win平台的,Mac不太了解.首先,你需要明白你安装Linux的目的,究竟是玩一玩耍酷,还是准备长期使用。如果是前者,我推荐可以在虚拟机下进行安装,因为这样维护成本低,也不会出现莫名其妙的错误,如果是后者,我的建议是双系统,可以很方便的回到Win的怀抱。但如果想要完整的学习Linux,在实机下装是必须的!

我这里先假设读者是准备长期使用Linux,那么你需要面对的是如下几个问题,关系到双系统的安装和能否成功启动.

  1. 启动类型是UEFI还是Legacy
  2. 是双硬盘还是单硬盘

首先,什么是UEFI？请参阅:wiki:UEFI,而大部分2013年暑假买到的电脑,基本都是UEFI的BIOS,而读者必须明白的是,UEFI模式下的引导十分负责复杂,稍有影响,就会导致你的WIndows 8无法正常引导(但是按照本文来操作,不太可能出现这种情况),不过鉴于这种情况太复杂,我还是建议读者使用Legacy模式安装(不过这样的话需要你全盘格式化,并且重新安装Win8). 具体启动类型的查看需.要进入到 BIOS里面,不同主板BIOS进入方式不同,开机是狂按F2或者ESC或者DELETE,(.如果是很 好看的图形化界面,那就是UEFI),然后找到BOOT MODE,检查以下是UEFI还是Legacy.

如果是Legacy,则不用担心这些问题.

其次,是多硬盘,还是单硬盘.如果你确信你的硬盘是2个及以上的话,那么恭喜你,第一个问题你完全不用考虑!无论哪一种方式安装,都会是轻松快捷,当然混合硬盘不属于双硬盘.

启动介质的准备

Linux的发行版有很多,但一般来说难易程度逼格是这样的debian/Ubuntu < Fedora/openSUSE < Arch/Gentoo,而最后面的2个还需要自己编译才能使用.

安装系统自然需要这个系统的介质,本文目前介绍的是Ubuntu Gnome 13.10,为什么选这个？因为Ubuntu目前可以说是人气最高(之一)的Linux发行版,BUG较少 比如说我在Fedora, openSUSE下都遇到了奇怪的问题,而且由于Ubuntu的高人气,很多问题都可以很方便的Google到,至于什么是Gnome,可以参阅:wiki:GNOME,选择GNOME是因为其定制性强,而且外表 虽然这完全不是重点可以带给你抛弃Win的喜悦,不选择Ubuntu原生的Unity,是因为其丑陋,而且会有很多莫名其妙的问题.

下载地址:Ubuntu-Gnome

而如何配置呢？刻成光盘么？完全没有必要,我们只需要将其烧录到U盘即可,在Win下下载一个ultraiso的试用版吧!然后选择[启动]--[写入硬盘镜像]烧录到U盘,这样启动介质就做好了!

硬盘分区

如果你是双硬盘,那么可以用一整块硬盘安装Linux,这样是最方便的。但是我相信大部分情况下你只会有一块硬盘,这个时候,可以将最后一个win下的分区作为Linux安装的空间,但如果空间很多,可以用过PQ或者DQ之类的软件分割出来一块空间,但是“数据无价,操作谨慎”,望读者注意.

2.1 显示

双显卡的配置问题

关于双显卡在linux下工作的问题,其实我这里说多了估计大家也没什么意思,在这里,我只向大家介绍N卡的双显卡(因为我只有N卡本,而且A卡驱动之中就有直接的设置).具体N卡双显卡的工作原理我不再赘述,大家可以参阅:wiki:Bumblebee

不过Ubuntu在这方面还比较简单,具体下来只需要以下几个命令：

$ sudo add-apt-repository ppa:bumblebee/stable
$ sudo apt-get update
$ sudo apt-get install bumblebee virtualgl linuxheaders-generic

具体什么是apt-get可以参阅:wiki:APT-RPM,如果你是Fedora系的,可以参阅:wiki:YUM

这几步做好之后,我们就成功的安装好了Bumblebee,现在,我们重启一下电脑,然后输入

$ lspci |grep VGA

返回如下信息,就代表你的双显卡配置成功了(电脑再也不会疯狂发热了

00:02.0 VGA compatible controller: Intel Corporation 3rd Gen Core processor Graphics Controller (rev 09)
01:00.0 VGA compatible controller: NVIDIA Corporation GK107M [GeForce GT 650M] (rev ff)"

注意第二项表示的是独立显卡,如果是ff就是代表关闭,如果是a1就是代表开启,如果想用独显运行程序,只需要输入

$ optirun XXX

分辨率设置问题

某些设备,可能处于某些原因,并不能匹配到合适的分辨率,或者像我一样,习惯了双屏输出,可是第二屏并没有合适的分辨率,这个时候,我们需要手动设置以下分辨率,运行如下命令即可

$ cvt 1440 9000

会返回如下的信息

# 1440x900 59.89 Hz (CVT 1.30MA) hsync: 55.93 kHz; pclk: 106.50 MHz
Modeline "1440x900_60.00"  106.50  1440 1528 1672 1904  900 903 909 934 -hsync +vsync

记录下来后面每个分辨率具体的数据,然后运行

$ xrandr --newmode "1440x900_60.00"  106.50  1440 1528 1672 1904  900 903 909 934 -hsync +vsync
$ xrandr --addmode VGA1 1440x900_60.00
$ xrandr --output VGA1 --mode 1440x900_60.00

注意,我这里输入的VGA1是的我第二屏,笔记本应该是LVDS1,具体可以输入xrandr查看.如果运行成功的话,分辨率就会变成你想要的样子,怎么样,比windows下简单多了吧 可能读者并没有遇到win下分辨率异常的情况,但是笔者遇到的时候必须在显卡驱动里面设置才行,非常麻烦

但是,这个设置只是暂时的,因为一关机设置又会变回原来的样子,稍微复杂一劳永逸的方法是修改xorg.conf,但那种方法有点点麻烦,这里介绍一个shell脚本的方法,什么是shell脚本,请看:Shell脚本入门

用一个正常的文本编辑器,输入：

#! /bin/bash
xrandr --newmode "1440x900_60.00"  106.50  1440 1528 1672 1904  900 903 909 934 -hsync +vsync
xrandr --addmode VGA1 1440x900_60.00
xrandr --output VGA1 --mode 1440x900_60.00

保存为xxx.sh,然后给他加上可以运行的权限 请参阅: 鸟哥的Linux私房菜:权限

$ chmod +x xxx.sh

以后,开机只要定位到这个sh文件的目录,然后运行

$ ./xxx.sh

就可以快速的修改分辨率了而且这种方法显得很Geek

3.2 虚拟机

准备一个win的虚拟机,因为有些时候还是会.离不开Win 很大程度上归结与宏福这边访问学校必须要挂VPN,然后这个VPN基于ActiveX,而且还有一些Linux下暂时没有替代的软件 比如说QQ,虽然Linux可以用Google的Hangouts,但应该很少人用吧.

虚拟机下有2个比较常用,一个是Oracle的免费VirtualBox,另一个是收费的VMware系列,但是在文档这里,我推荐的是:VMware Player.有2个原因,第一个是VMware Player是免费的,而且最新版允许新建虚拟机,第二个是因为VirtuaBox不够好用,很多东西都很恶心.

VMware Player下载地址

发现下载下来的文件是.bundle,那我们如何安装呢？只需要给其加上可以执行的属性即可.如何加权限?请看: 分辨率设置问题

安装过程一路下一步即可,这里不再赘述.

需要注意的是,虚拟机本身不带有系统,所以需要你自己下载一个系统镜像,这里不附带下载地址.整个虚拟机安装系统不会差太多,但如果你想用你自己备份好的GHO来恢复,需要注意之前要用分区软件重建虚拟磁盘分区表,并且设置为活动(Active),否则是无法引导的.

安装好系统之后,在虚拟机内部装好VMware Tools,具体在虚拟机的菜单里有Install VMware Tools这一项,完成这一项,虚拟机才可以自适应窗口,并且毫无压力的与主机进行数据交换.为了不影响主机的活动,可以将虚拟机发送到其他的工作区,这样就可以瞬间在Linux和Win之间切换,毫无压力.

3.3 c/c++开发环境设置

很多的OJ都是在Linux下评测程序的,所以设置一个c/c++开发环境是必要的.而Ubuntu里面默认只有gcc 这里的gcc狭义指gcc这个程序,不含g++和其他库,真正意义上的gcc请看:wiki:GCC, 所以首先我们需要下载一些开发工具.

$ sudo apt-get install build-essential

这个时候,我们可以写一个简单的c程序测试一下,这里,我强烈推荐Vim这一款编辑器 如果要深入的学习Linux,以后和Vim打交道的次数很多很多, 学习Vim,请看: 简明 Vim 练级攻略.在Vim里写好一个简单的Hello,world,保存为hello.c

#include
int main() {
	printf("Hello,world!\n");
	return 0;
}

还记得编译过程吗？请看如下的命令

$ gcc -E hello.c -o hello.i #预处理
$ gcc –S hello.i –o hello.s #编译
$ gcc –c hello.c –o hello.o #汇编
$ gcc hello.o -o Hello      #连接
$ ./hello
  Hello,world!

其实一般情况下不用这么麻烦,只需要:

$ gcc hello.c -o hello

编译器我们已经准备好了,其实到这里一个开发环境已经做好了,但如果想更轻松的完成任务,我们还需要一个(集成开发环境) 什么？IDE是什么意思？编辑器 编译器和集成开发环境

Linux以下一般常用的是Code::Blocks,这样安装即可:

$ sudo apt-get install codeblocks

只要熟悉Win里面CodeBlocks的操作,怎样使用不是难事,不过尴尬的是,其自带的终端不能复制粘贴,我们可以将其替换为普通的Terminal.只要Setting==>Environment...==>General Setting==>Terminal to lanuch console programs, 用“gnome-terminal -t $TITLE -x”替换掉“xterm -T $TITLE -e”就搞定了

Java开发环境设置

首先在Oracle的官网上下载最新的JDK

首先,解压安装

$ sudo mkdir /usr/lib/jvm
$ sudo tar zxvf jdk-7u51-linux-x64.tar.gz -C /usr/lib/jvm
$ cd /usr/lib/jvm/
$ sudo mv jdk1.7.0_51/ jdk-7-sun

然后修改环境变量

$ vim ~/.bashrc #还记得Vim么?
#添加
export JAVA_HOME=/usr/lib/jvm/jdk-7-sun
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib
export PATH=${JAVA_HOME}/bin:$PATH
#保存,输入以下内容使其生效
$ source ~/.bashrc

最后,配置默认jdk版本

$ sudo update-alternatives --install /usr/bin/java java /usr/lib/jvm/jdk-7-sun/bin/java 300  
$ sudo update-alternatives --install /usr/bin/javac javac /usr/lib/jvm/jdk-7-sun/bin/javac 300  
$ sudo update-alternatives --install /usr/bin/jar jar /usr/lib/jvm/jdk-7-sun/bin/jar 300   
$ sudo update-alternatives --install /usr/bin/javah javah /usr/lib/jvm/jdk-7-sun/bin/javah 300   
$ sudo update-alternatives --install /usr/bin/javap javap /usr/lib/jvm/jdk-7-sun/bin/javap 300

测试以下,是否成功

$ java -version
java version "1.7.0_51"
Java(TM) SE Runtime Environment (build 1.7.0_51-b13)
Java HotSpot(TM) 64-Bit Server VM (build 24.51-b03, mixed mode)

至于IDE,Eclipse,还是去官网下载最靠谱吧!解压即可用！

3.5 web开发环境配置

要开发WEB,我们首先就要配置WEB环境,比较著名的环境就是LAMP,参阅wiki:LAMP,而在Linux下安装,十分简单,只需要很少的命令就可以完成所有的事情!在这里可以尽情的鄙视Windows

$ sudo apt-get install tasksel
$ sudo tasksel install lamp-server

Tasksel是一个很方便的Linux下安装服务器程序的软件,直接安装LAMP组件,则可以自动帮你安装apache,MySQL,PHP,并且可以很方便的解决所有依赖问题,十分方便!

默认的www目录(就是网页存储的地方啦),在/var/www,但是该目录的所有者不是当前用户,所以我们需要修改其owner,这样才可以十分方便的配置网页. 还记得目录权限吗？鳥哥的 Linux 私房菜:Linux 的檔案權限與目錄配置

$ sudo chown nil /var/www #这里的nil就是当前用户名咯

检验apache是否安装好了,可以访问http://localhost,出现It works就代表Apache是完美的,接着在/var/www里面添加index.php

	
<?php
	phpinfo();
?>

然后访问http://localhost/index.php,如果出现php的信息,那么php也是正常的!

不过安装好了LAMP组件之后,对于新手来说,如何配置数据库啊什么的还是一件难事,这里,我推荐安装webmin这个组建,可以很方便的配置,去官网下载,安装即可,之后就可以去https://localhost:10000/,输入当前用户名和密码就可以管理啦!

至于前端编辑器,笔者推荐的是Sublime Text + Chrome的组合,十分方便!下面的就是Sublime Text的调试攻略,首先就是安装了～

$ sudo apt-get install sublime-text-installer

还是一条命令,十分简单,其实单独的文本编辑器谁都可以,但是ST的厉害在于她的性感以及其强大的插件,在ST里面CTRL+~打开console栏,输入命令,安装插件管理程序Package Control

import urllib.request,os; pf = 'Package Control.sublime-package'; ipp = sublime.installed_packages_path(); urllib.request.install_opener( urllib.request.build_opener( urllib.request.ProxyHandler()) ); open(os.path.join(ipp, pf), 'wb').write(urllib.request.urlopen( 'http://sublime.wbond.net/' + pf.replace(' ','%20')).read())

之后就可以在Package Control中安装插件,CTRL + P 输入install,就可以搜索安装各种插件了,这里推荐几篇文章,这里不再赘述 Sublime Text 3 插件
